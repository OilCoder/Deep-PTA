Deep PTA

A neural network that does the senior interpreter's job: it reads the Δp + Bourdet-derivative curve (log-log) of a pressure transient test and diagnoses the reservoir model, the boundary, and the key parameters.

Trained 100% on synthetic data from a certified analytical engine (Laplace solutions + Stehfest inversion), verified against published Bourdet/Gringarten type curves and AnaFlow. No commercial simulator.

The honest result

Headline numbers are balanced accuracy (mean per-class recall) on a class-stratified test set — raw accuracy lies under class imbalance. Two test sets are reported: an in-distribution holdout (headline) and a held-out high-storage extrapolation set the model never trains on.

Headline is a ResNet-1D + PatchTST ensemble. The boundary "0.84" was a mirage (old test ~70% infinite-acting). The real lift came from data scale: a GPU-batched engine — validated against the certified CPU engine to ~4e-7 — made training on millions of curves cheap, raising reservoir balanced accuracy 0.51→0.65 and nearly halving the parameter MAE. Homogeneous reservoir recall went 0.12→0.59. A 2× larger model matched the small one, so capacity was not the bottleneck — data was. The residual ceiling is the physical homogeneous↔infinite-fracture non-uniqueness.

What's inside

  • Analytical engine — homogeneous, double porosity, infinite/finite fractures; infinite / sealing-fault / constant-pressure / closed boundaries.
  • GPU-batched engine — a PyTorch port of the analytical engine validated against the certified CPU engine to ~4e-7; generates millions of curves in minutes, preserving the data distribution.
  • Synthetic generator — realistic noise, drift, truncation, domain-aware validity filter; a hybrid C_D split (in-distribution holdout + extrapolation).
  • Two models + ensemble — multi-task ResNet-1D and a hand-built PatchTST-style Transformer over a 3-channel log-log representation, fused by an ensemble.
  • Reproducible training — Optuna HPO on balanced accuracy, early stopping, TensorBoard; one entrypoint.
  • App + narrator — Gradio CSV→diagnosis with a fitted-curve overlay and an optional Claude-based narrator.

Method & rigor

The accuracy story — why the metric was misleading, and a quantitative test (error concentration ratio) of whether residual error is physical non-uniqueness or pipeline — is documented in the project's method reports.

Built with Python · NumPy/SciPy · PyTorch · Optuna · Gradio. Single-phase oil MVP; real-case validation is future work.
