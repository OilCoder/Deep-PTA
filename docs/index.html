Deep PTA

A neural network that does the senior interpreter's job: it reads the Δp + Bourdet-derivative curve (log-log) of a pressure transient test and diagnoses the reservoir model, the boundary, and the key parameters.

Trained 100% on synthetic data from a certified analytical engine (Laplace solutions + Stehfest inversion), verified against published Bourdet/Gringarten type curves and AnaFlow. No commercial simulator.

The honest result

Headline numbers are balanced accuracy (mean per-class recall) on a class-stratified test set — raw accuracy lies under class imbalance.

HeadBaseline (raw / balanced)Current (balanced)
Reservoir (4 classes)0.42 / 0.430.51
Boundary (4 classes)0.84 / 0.610.70

The boundary "0.84" was a mirage: the old test set was ~70% infinite-acting, so the metric rode the majority class. On a balanced test the model genuinely improves the classes that matter — sealing fault 0.35→0.59, constant pressure 0.40→0.60, closed 0.71→0.85 — and homogeneous reservoir recovers from 0.12 to 0.33.

What's inside

  • Analytical engine — homogeneous, double porosity, infinite/finite fractures; infinite / sealing-fault / constant-pressure / closed boundaries.
  • Synthetic generator — realistic noise, drift, truncation, domain-aware validity filter; disjoint C_D bands for train/val/test.
  • Two models — multi-task ResNet-1D and a hand-built PatchTST-style Transformer over a 3-channel log-log representation.
  • Reproducible training — Optuna HPO on balanced accuracy, early stopping, TensorBoard; one entrypoint.
  • App + narrator — Gradio CSV→diagnosis with a fitted-curve overlay and an optional Claude-based narrator.

Method & rigor

The accuracy story — why the metric was misleading, and a quantitative test (error concentration ratio) of whether residual error is physical non-uniqueness or pipeline — is documented in the project's method reports.

Built with Python · NumPy/SciPy · PyTorch · Optuna · Gradio. Single-phase oil MVP; real-case validation is future work.
